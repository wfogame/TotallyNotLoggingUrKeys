# Workforce Productivity Monitor

> **Internal tool — private repository. Do not share outside the organisation.**

A lightweight, tamper-resistant employee productivity monitoring system for company-owned Linux machines. Tracks keystroke counts, mouse activity, active/idle time, and per-window application usage. Includes a remote command channel for authorised managers and a tamper watchdog that ships alerts to a central log server.

---

## ⚠️ Ethical Standards & Disclaimer

This tool was built with workplace transparency and employee dignity as non-negotiable principles. The following standards **must** be upheld at all times. Failure to comply may expose the organisation to legal liability and constitutes a misuse of this software.

### Mandatory Requirements

- **Informed Consent**
  Every employee whose machine runs this software must be explicitly informed, in writing, before monitoring begins. A signed acknowledgment (or equivalent record) must be retained by HR. Covert deployment on any machine is strictly prohibited.

- **Company-Owned Devices Only**
  This software must only be installed on machines owned by the company. Installation on personally owned hardware — including bring-your-own-device (BYOD) setups — is prohibited under all circumstances.

- **Purpose Limitation**
  Data collected by this tool may only be used for legitimate workforce management purposes (e.g., productivity review, compliance auditing). It must not be used for harassment, retaliation, or any purpose unrelated to employment management.

- **Data Minimisation**
  This software records **keystroke counts only** — it does not log what was typed. No screenshots are taken. No audio or video is captured. No browsing history is recorded. Scope must not be expanded without legal review.

- **Access Control**
  Log files and remote access credentials must be restricted to authorised personnel only (e.g., direct managers, HR, compliance officers). Sharing credentials or logs with unauthorised parties is prohibited.

- **Retention Limits**
  Log data should not be retained beyond what is operationally necessary. A retention policy must be defined and enforced before deployment.

- **Legal Compliance**
  Deployment must comply with all applicable laws and regulations in the jurisdiction(s) where the organisation operates. This includes but is not limited to:
  - **USA:** Electronic Communications Privacy Act (ECPA), state-level wiretapping laws
  - **EU / UK:** GDPR / UK GDPR — employee monitoring requires a lawful basis, a legitimate interest assessment, and transparent notice
  - **Other regions:** Consult local legal counsel before deployment

  > **This tool does not constitute legal advice. The deploying organisation is solely responsible for ensuring lawful use.**

### What This Tool Does NOT Do

| Capability | Status |
|---|---|

## Repository Contents

| File | Description |
|---|---|
| `productivity_monitor.py` | Core monitor — keystrokes, mouse, time, per-window activity |
| `tamper_watchdog.py` | Watchdog — detects tampering, ships alerts to syslog server |
| `remote_server.py` | Runs on employee machines — accepts manager commands over TLS |

| `install.sh` | User-level installer (legacy, superseded by install_system.sh) |
| `uninstall_system.sh` | Clean removal of all components |
| `productivity-monitor.service` | Systemd service unit (generated by installer) |
| `productivity_monitor_docs.docx` | Full system documentation and user guide |

## Quick Start

> Full instructions are in `productivity_monitor_docs.docx`.

**Deploy on an employee machine (IT admin):**
```bash
sudo bash install_system.sh
```

**Connect remotely from a manager's machine:**
```bash
python3 remote_client.py --host <employee_ip> --token <secret>
```

**Check everything is running:**
```bash
systemctl status tamper-watchdog
systemctl status 'productivity-monitor@*'
```
